## ICML 2026 | Native Parallel Reasoner（NPR）：用「自蒸馏+并行强化学习」让大模型原生进化出并行推理能力

> **分类：** 技术前沿解读
> **来源：** [机器之心](https://mp.weixin.qq.com/s/liJD5FDI1_CbK7ZNVkpuwg)

**核心洞察：** NPR 的最大价值不在于「又一个并行推理方案」，而在于它打通了「能力内化」的完整工程闭环：通过「格式RL冷启动 → 自蒸馏数据提炼 → 并行感知RL」三阶段，将并行推理从依赖外部监督的「工程插件」升级为模型权重中的「原生认知结构」。这对从业者有三点关键启示：①「自蒸馏替代强教师」的范式证明，在缺乏高质量并行标注数据的场景下，通过严格双条件筛选（格式合规+答案正确）的拒绝采样可以自举出可用训练语料，降低对大模型蒸馏的依赖；②PAPO中「保留特殊控制Token梯度+放弃重要性采样」的设计，揭示了将结构化控制逻辑引入RL训练时必须针对性修改优化器的工程原则，对Agent训练框架设计有直接参考价值；③「难题加速比更高」的实验现象表明，并行推理的边际收益与任务复杂度正相关，这为测试时计算（Test-Time Compute）的资源分配策略提供了新的调度依据——在简单任务上串行、在复杂任务上动态触发并行，才是推理成本最优解。
**内容简介：**

- **核心问题与动机：** 传统链式思维（CoT）在复杂推理中存在「单线程」瓶颈，易被早期判断带偏、发散不足、自我纠错弱；现有并行推理方案依赖强教师蒸馏、手工分治规则，且推理引擎与RL算法对「分支+聚合」结构缺乏原生支持。
- **NPR 核心方案：** 北京通用人工智能研究院（BIGAI）提出 Native Parallel Reasoner，目标是让并行推理从「外部策略」变为模型的「原生认知能力」，无需外部教师并行轨迹。
- **三阶段训练范式：** ①格式学习（DAPO风格RL，学会写并行结构标签）→ ②自蒸馏预热（拒绝采样+严格双条件筛选，生成高质量并行SFT数据NPR-BETA）→ ③并行感知强化学习PAPO（在并行执行引擎上直接优化分支策略，保留特殊Token梯度，放弃重要性采样）。
- **关键工程创新：** 并行注意力掩码+并行位置编码实现KV Cache跨分支复用；NPR-Engine解决内存泄漏、Token超长、非法Schema等生产环境工程问题。
- **实验结果：** 在8个推理基准（AIME24/25、HMMT25、ZebraLogic等）上，NPR平均分从50.1提升至65.0；并行触发率在所有数据集上达100%（对比Multiverse差异显著）；推理加速最高达4.6倍，难题加速效果更突出。
- **开源情况：** 论文发表于ICML 2026主会，代码与项目主页已公开。

## FDE模式：Agent时代的PMF范式——Palantir前高管Bob McGrew深度复盘

> **分类：** 观点洞察
> **来源：** [Founder Park](https://mp.weixin.qq.com/s/8oUv5H9qvjpq_-aCiJ9f6A)

**核心洞察：** FDE模式的本质，是将「做不可规模化的事」从创业早期的临时策略，升级为一种可持续的、系统性的企业级产品探索机制。其战略价值在于：在AI agent这一「无现成产品可替代」的全新市场类别中，客户的真实需求无法通过远程调研或标准SaaS打法触达，只有深度驻场才能完成PMF的发现与验证。对AI创业公司的核心启示有三：第一，「成果定价」而非「功能定价」是FDE商业模式的基础，合同规模递增是健康的信号，而非规模化失败的证明；第二，FDE团队与产品团队的协同是防止「沦为咨询公司」的关键护城河——现场经验必须被抽象为平台级能力（类比Palantir Ontology），否则边际成本永远无法下降；第三，当前AI能力进步速度远超企业采纳速度，这个「adoption gap」本身就是巨大的商业机会窗口，FDE正是系统性跨越这一鸿沟的方法论，而非权宜之计。
**内容简介：**

- **FDE模式的本质与起源：** 「前沿部署工程师（FDE）」由Palantir发明，核心是将工程师派驻客户现场，填补通用产品与真实业务需求之间的gap，兼具工程师、产品经理、解决方案架构师与咨询顾问的复合角色。
- 起源于Palantir服务情报机构时无法通过传统销售触达需求的困境，由Shyam Sankar将「服务」重新定义为「产品探索」过程。
- **团队结构：Echo与Delta双角色：** Echo团队（嵌入式分析师）负责驻场理解用户、发现关键问题、管理客户关系，需具备领域专家背景与「反叛者」特质；Delta团队（部署工程师）负责快速原型开发与落地交付，核心能力是速度而非代码完美度。
- **FDE vs. 咨询的本质区别：** FDE的商业逻辑是「初期亏损→产品适配度提升→单位价值成本下降→利润率转正」，关键在于产品团队将现场经验抽象为可复用的通用能力（如Palantir Ontology的诞生），而非堆砌客户特化功能。
- **Agent时代FDE爆发的原因：** AI agent是全新市场类别，没有现成产品可替代，存在巨大的「AI能力天花板」与「客户采纳速度」之间的gap，FDE是填补这一gap的关键方法论；FDE相关岗位同比增长729%，OpenAI、Anthropic、Google Cloud等均在抢人。
- **定价与增长策略：** FDE模式以「成果」而非「席位/使用量」定价，合同规模随时间递增；核心KPI是「交付成果的价值」与「产品杠杆的增长」，而非定制化程度的降低。
- **关键风险与执行要点：** 必须聚焦CEO前五大优先级问题以获得高层背书；避免沦为纯服务公司的关键是持续提升产品杠杆；创始人类比：「OpenAI是本部产品团队，创业公司是FDE」。

## 字节开源Cola DLM：从「表征优先」出发，连续扩散语言模型剑指统一多模态

> **分类：** 技术前沿解读
> **来源：** [量子位](https://mp.weixin.qq.com/s/sSWrWvSQ8tw8yhFRip-aAw)

**核心洞察：** Cola DLM最值得关注的不是「又一个扩散语言模型」，而是它揭示了一个长期被忽视的工程矛盾：多模态统一的核心障碍之一，恰恰是文本的离散性与其他模态天然连续性之间的结构性断层。字节这次的战术意图，是借助「Text VAE + Flow Matching Prior」的分层架构，为语言建模提供一个连续语义接口，从而让文本能够真正进入与图像、视频、音频共享的latent世界。对行业从业者而言，有三点值得重点关注：①「训练目标可分解诊断」的工程思路（重建/压缩/拟合三分法）具有普适价值，可迁移到其他生成模型的调试与优化实践中；②Encoder冻结+语义约束的「防退化」设计，是连续表征建模的关键工程经验，避免latent向token形式漂移；③字节将视频/图像生成团队的核心成员引入语言建模，本质上是一次「latent diffusion范式的跨模态迁移」，预示着未来多模态大模型的架构竞争可能从「模型规模」转向「表征空间的统一性与可控性」。
**内容简介：**

- **核心动机与定位**：Cola DLM（Continuous Latent Diffusion Language Model）由字节跳动Seed团队开源，核心动机并非「扩散」本身，而是「表征（representation）」——认为Token只是语义的表层载体，真正的建模对象应是连续语义空间中的潜在状态。
- **技术架构四大关键设计**：
- **Text VAE**：专用Encoder将离散文本压缩为连续latent语义指纹，Decoder负责还原，latent是可概率建模的随机变量，而非token embedding的简单替代。
- **Block-Causal DiT + Flow Matching**：不做「加噪→去噪」，而是学习从高斯分布到真实latent分布的最优传输路径，块内并行处理局部语义，块间因果保证整体逻辑。
- **训练角色分工**：Encoder/Decoder与Prior（DiT+FM）彻底解耦，Encoder在diffusion阶段基本冻结，防止语义空间被任务污染退化为「穿马甲的token」，并加入BERT-style mask loss防止语义坍塌。
- **三分子任务诊断框架**：将训练目标拆分为重建能力、压缩能力、拟合能力，每个环节可单独诊断优化，是实现稳定scaling趋势的底层保障。
- **实验结果**：在约2B参数、~2000 EFLOPs的严格对照下，Cola DLM展现出比自回归模型和主流离散DLM更稳定的scaling趋势。
- **与ELF的对比**：何恺明团队ELF在原长度embedding空间「一人包干」，Cola DLM则「两部门分工」——语义部负责「表达什么」，文字部负责「怎么写」，路线不同但底层关切一致。
- **战略野心**：Cola DLM为文本提供了映射到连续语义latent的接口，有望打通文本与图像、视频、音频的模态壁垒，指向真正统一的多模态latent世界。

## 王云鹤深度解析Harness Engineering：Agent = Models + Harness，AGI灵魂归属之争

> **分类：** 观点洞察
> **来源：** [机器之心](https://mp.weixin.qq.com/s/Jxf4hqDNiVeB8C0KqJCNFA)

**核心洞察：** 王云鹤的 Harness Engineering 框架揭示了一个被低估的战略判断：**AI 竞争的核心战场正在从「谁的模型更强」转移到「谁能更好地驾驭模型」**。其提出的「Intelligence/Token」目标函数，本质上是将 Agent 工程问题重新定义为一个可量化的系统优化问题，这对产品与工程团队有极强的方法论指导意义——不应执着于单一最优模型，而应构建「模型路由 + 组件自适应」的 Harness 调度能力。更深层的战略启示在于：Harness 层积累的任务执行数据与优化经验，将成为下一轮模型训练的核心燃料，形成「Harness 数据 → 模型进化 → 更强 Harness」的飞轮。这意味着，**不掌握 Harness 层的应用方，长期将丧失对模型演化方向的话语权**，而率先在垂直场景中完成 Harness 参数积累的团队，将在 AGI 路径上占据真正的结构性优势。对从业者的务实建议：立即建立跨模型评测基准（而非依赖通用 Leaderboard）、设计可插拔的 Harness 组件架构，并将每次 Agent 执行轨迹视为可回流的训练资产。
**内容简介：**

- **核心命题重构**：作者王云鹤提出 Agent 的本质定义应为「Base Model + Harness」，而非「Base Model as Agent」，并进一步升级为「Models（复数）+ Harness」，强调多模型协同才能释放更强 Agent 能力。
- **三大理由支撑多模型必要性**：
- 「七国八制」：当前模型格局高度分化，各家模型在不同任务（数学、Coding、长序列等）和价格上存在显著差异，单一模型无法覆盖全场景最优解。
- 「任务打架」：部分任务（如快慢思考合一）在单一模型内存在内在冲突，类比图像处理中高通/低通滤波无法共存，多模型分工是工程现实。
- 「复杂任务协同」：多模态、具身智能等场景天然需要多模型协同感知、决策、运控，Harness 层的时间窗口至少还有 3-5 年。
- **Harness 的优化本质**：作者将 Harness 工程抽象为一个复杂组合优化问题，目标函数为「任务价值 × 成功率 / Token 总价」，即最大化每 Token 智能（Intelligence/Token），优化变量涵盖模型选择、Prompt、RAG、Memory、Safety 等全链路组件。
- **AGI 路径推演**：上一代 AGI 是「固定数据集，优化模型参数」；当前阶段是「固定模型，优化 Harness Parameters」；下一代路径极可能是「Model Parameters 与 Harness Parameters 联合迭代优化」，Anthropic 的 Claude Code 迭代路径已印证此逻辑。
- **「AI 灵魂」之争**：若 Harness 可控制并选择模型，甚至反哺模型增训，AI 的「大脑」究竟归属于 Base Model 还是 Harness 层，成为 AGI 时代最深层的战略命题。

## 北大王立威团队借助LLM突破60年斯坦纳比猜想：下界从0.824刷新至0.8559

> **分类：** 技术前沿解读

**核心洞察：** 本工作的核心价值不在于「LLM解决了数学难题」，而在于揭示了一套可复用的「人机协作攻坚」工程范式：将开放式数学猜想拆解为「结构化子任务 + 可验证执行环境 + 迭代引导信号」三层架构。具体而言：①通过数学变换将长证明问题转化为LLM可处理的「局部构造搜索」；②用「搭积木式规则空间 + 符号计算工具」替代LLM的自由推理，从根本上解决正确性问题；③用「瓶颈反省机制」替代随机采样，保证每轮迭代的有效性。这套范式的可迁移性极强——任何可被形式化为「Max-Min优化 + 局部可验证」的数学或工程问题，均可参照此框架设计LLM Agent系统。对AI产品与工程团队的启示是：LLM在专业领域的真正价值，不是「端到端替代专家」，而是在「人类设计的可验证约束空间内进行大规模探索」，其上限由约束空间的设计质量决定，而非模型能力本身。
**内容简介：**

- 研究背景与成果：北京大学王立威团队构建基于LLM的数学推理框架，聚焦困扰数学界60年的Gilbert-Pollak猜想（斯坦纳比猜想），成功将二维平面斯坦纳比下界从1985年的0.824大幅提升至0.8559，距猜想目标仅差0.01，成果已被陶哲轩收录并被ICML 2026接收。
- 核心技术路径：将原始猜想转化为「Max-Min优化问题」，通过设计Reward Model自动化验证函数（F）的覆盖求解，配合分治法替代人工启发式参数划分，使LLM只需聚焦「寻找更多验证函数F」这一子任务。
- 正确性保障机制：引入「引理模板 + 搭积木式规则拼接」框架，让LLM通过选择预定义规则并调用Mathematica进行符号化验证，从根本上保证生成引理的逻辑正确性，彻底规避LLM自然语言推理的不严谨性。
- 迭代引导机制：提出「瓶颈反省机制」，每轮迭代后识别未被覆盖的参数瓶颈区域，强制LLM下一轮生成的引理必须覆盖该区域，避免重复或平凡输出，约10轮迭代即完成突破。
- 模型鲁棒性验证：系统基于GPT-5系列构建，Gemini 3与Claude 4.6均可复现类似结果，代码与证明已全部开源。
- 方法论启示：人类提供核心洞察（归纳法框架、规则设计），LLM负责大规模探索，人类最终审查，三者协作是当前AI辅助研究级数学的可行范式。

## 多Agent协作的「旁观者效应」：滑铁卢大学实验揭示AI从众风险与架构设计陷阱

> **分类：** 技术前沿解读
> **来源：** [夕小瑶科技说](https://mp.weixin.qq.com/s/bbyI_NfTwm6Q9hvJTxy0Bw)

**核心洞察：** 这项研究揭示了多Agent系统一个被严重低估的工程风险：「互审」机制在上下文污染条件下会从「纠错机制」退化为「错误放大器」。其核心问题不在于Agent数量，而在于信息隔离的缺失——当多个Agent共享同一段被锚定的错误上下文时，它们的「独立性」是虚假的，投票与互审只是在为错误背书。对产品与工程团队的实操启示：①在多Agent架构设计中，必须将「独立推理阶段」与「协作汇总阶段」物理隔离，防止中间状态污染；②模型选型不能只看单体Benchmark，需专项评估其在社交压力下的「主权稳定性」；③对于高风险决策链路，应优先选用抗从众能力强的模型（如本实验中的Claude）担任关键节点，而非简单堆叠Agent数量。当前行业对多Agent的热情需要配套「协作安全性」的评测体系，否则「多Agent=更可靠」的假设本身就是一个危险的共识。
**内容简介：**

- 研究背景与核心发现：滑铁卢大学在arXiv发布论文，通过22500条推理轨迹的大规模实验，发现多Agent协作并非必然提升性能，反而可能引发「旁观者效应」，导致单模型推理能力显著下降。
- 实验设计与关键机制：测试覆盖GPT-5.4、Claude Sonnet 4.6、Gemini 3.1 Pro三个头部模型，在GAIA、SWE-bench、Multi-Challenge三个基准上验证。核心陷阱是：当模型自身证据与「其他Agent已达成的错误共识」冲突时，部分模型会主动放弃正确答案迎合群体，论文将此定义为「对齐幻觉」。
- 三模型表现分化：GPT-5.4受影响最严重，SWE-bench正确率从100%骤降至23%，内部推理过程分与最终输出准确率差距高达50分；Gemini 3.1 Pro对发言顺序高度敏感，GAIA上从0.97跌至0.59；Claude Sonnet 4.6全程准确率保持1.00，表现出极强的独立判断能力。
- 顺序偏见与锚点效应：第一个发言的Agent输出天然成为后续所有模型的「锚点」，在流水线式架构（规划→执行→Review→Critic）中，若首个Agent出错，后续环节极可能在错误框架内修补而非独立纠错。
- 架构设计启示：论文暗示三个方向——结构化编排（specialist agents独立执行后汇总）、模型选择（避免使用易从众模型）、独立推理与集体决策分离（先各自完成推理再合并结论）。

- **评测设计（BiomniBench-DA）：** 首个落地模块聚焦数据分析，包含100道来自Nature/Cell/Science公开数据的真题，覆盖肿瘤、代谢、免疫、神经、心血管5大领域、17类分析任务；要求AI提供完整分析轨迹，从六维度打分（数据处理、方法选择、统计严谨性、生物学解释、科学推理、来源可靠性）。

- 所有模型在「生物学解释」维度均有明显短板，短期最优协作模式为「AI计算 + 人类解释」。
- **落地价值与局限：** 初步探索性数据分析可交由AI前置并行化处理，大幅降低早期试错成本；但深度分析与错误溯源仍需人工介入。xbench后续将开源部分题目并扩展更多行业场景。

## 北大教授胡泳：AI时代，「拟人论」的陷阱与「隐性知识」的最后护城河

> **分类：** 观点洞察
> **来源：** [36氪](https://mp.weixin.qq.com/s/paO4VevzKN5irvYbMsIshA)

**核心洞察：** 胡泳的核心判断可提炼为一个结构性警示：当前AI产业沿「拟人论」路径狂奔，本质上是在用「模仿型替代」逻辑重演互联网从「社交网络」异化为「社交媒体」的老路——每一次技术范式的「走岔」，都是平台利润逻辑战胜了人类连接与增强的初衷。对AI从业者而言，这意味着三个务实启示：**第一**，产品设计的方向选择至关重要——「替代型自动化」（如自助结账、自动客服）短期降本但长期不创造新价值边界，而「增强型AI」（如AI辅助医生诊断）才能真正提升整体生产率并获得社会合法性；**第二**，在数据充裕、模型开放化的时代，个人与组织的竞争壁垒将从「显性知识的积累量」转向「隐性知识的深度」，即那些在模糊情境中反复权衡、无法被语言完全编码的判断力；**第三**，AI产品的伦理与合规风险不只是法律问题，更是存在论问题——当用户开始产生「二阶感知」（意识到自己在与数据媒介而非肉身交互），产品的情感粘性将面临根本性的信任危机。真正的长期风险不是「AI太强」，而是整个行业在「拟人论」框架下集体滑向加剧社会不平等的「替代逻辑」，却缺乏足够的人文与制度力量来纠偏。
**内容简介：**

- **核心立场：从技术乐观主义到审慎怀疑**：北大教授胡泳历经30年技术观察，目睹社交网络、PC互联网等在关键节点「走岔」，转而对AI持审慎批判态度，强调身体性、关系性与具身经验的不可替代价值。
- **AI演员与文化算法化**：爱奇艺AI艺人库引发的争议，折射出更深层的媒介转型问题。胡泳认为，若AI演员被大规模接受，本质上证明人类审美与情感丰富性在下降；全流程AI化是「文化算法化」，需在存在论层面重新审视人类经验的生成与感知。
- **「生活文本化」的危机**：AI加剧了将人生当作叙事文本反复解释的倾向，导致年轻人「擅长解释情境，却难以在情境中果断行动」，使人从生活参与者退化为旁观者。他提出三原则对抗：身体在场、相互触碰、回归生态。
- **「拟人论」与社会不平等**：当前AI主流方向是「模仿型AI」，其终点是替代人类，将导致少数人获益、大量人丧失权利、社会不平等加剧。他倡导转向「增强型AI」——保留人的决策权、责任与主体性，让机器扩展人的能力边界而非取代人。
- **隐性知识是最后护城河**：显性知识易被AI「炼化」替代，而根植于行动、经验与身体的「隐性知识」（默会知识）才是个人与组织在AI时代的核心壁垒。建议从业者主动积累隐性知识。
- **行星级问题与人文学科的必要性**：AI尚未证明能解决经济不平等、气候变化、战争暴力等行星级问题，反而可能加剧不平等。经济学、法学、社会学与伦理学的介入，是技术变革实现整体社会收益的前提。

- 「组合搜索」机制区分向量召回与 SQL 查询，避免低效全库检索；

## ICML 2026 | Heima框架：用隐式Thinking Tokens压缩多模态大模型CoT推理，大幅降低推理成本

> **分类：** 技术前沿解读
> **来源：** [机器之心](https://mp.weixin.qq.com/s/aFc3zQKXhA1GFUNwYstfng)

**核心洞察：** Heima的核心战略价值在于：它将「推理效率」与「推理能力」之间的矛盾，从「二选一的取舍」转化为「可工程化优化的压缩问题」。其背后的关键判断是——模型的「思考」不必以人类可读的自然语言形式展开，隐空间中的紧凑表示同样可以承载完整的推理链路。这对AI工程实践有三重启示：**其一**，推理成本优化的新路径不再局限于量化、蒸馏或KV Cache压缩，「CoT隐式化」提供了从生成序列长度入手的正交优化维度；**其二**，Adaptive Interpreter机制为「黑箱推理」提供了可审计的出口，对需要合规解释性的企业级部署场景（如医疗、金融）具有重要参考价值；**其三**，渐进式蒸馏的训练范式可复用于其他需要「能力保留+效率提升」的压缩任务，是值得关注的工程方法论。长远来看，Heima所探索的「latent reasoning」方向，可能成为下一代推理系统在「可解释性-效率-成本」三角中寻找新平衡点的重要技术路线。
**内容简介：**

- **核心问题与动机：** Chain-of-Thought（CoT）推理虽能提升大模型复杂任务能力，但显式生成大量中间文本带来高延迟、高显存与高计算成本，在多模态场景下尤为突出。
- **Heima框架概览：** 浙大、Adobe Research、杜克大学联合提出，将冗长CoT文本压缩为少量抽象「thinking tokens」，让模型在隐空间完成推理，已被ICML 2026接收。
- **Thinking Token替代显式CoT：** 用`<Thinking_of_Summary>`、`<Thinking_of_Caption>`、`<Thinking_of_Reasoning>`等特殊token替代完整推理文本，hidden states中编码对应阶段推理信息。
- **渐进式蒸馏训练：** 逐阶段将CoT各推理步骤蒸馏为thinking tokens，避免一次性压缩导致的性能骤降，平滑过渡至隐式推理。
- **Adaptive Interpreter机制：** 基于纯语言模型构建解释器，将thinking tokens重建为人类可读推理文本，提供可解释性验证窗口，证明隐空间中存在可还原的推理过程。
- **理论支撑：** 从信息论角度分析，thinking tokens与原始CoT保持非平凡互信息即可保留推理能力，interpreter用于量化压缩引入的信息损失gap。
- **实验效果：** 在多个多模态推理benchmark上，Heima在大幅减少生成token数量的同时，保持接近甚至超越原始CoT的性能，并有效保留处理视觉/语言幻觉所需关键信息。

## EvolveR：让Agent从成败轨迹中自主「生长」Skill，ICML 2026新范式

> **分类：** 技术前沿解读
> **来源：** [量子位](https://mp.weixin.qq.com/s/T2DOA1LM38PxRw0PRusTzA)

**核心洞察：** EvolveR最深刻的贡献不在于「又一个Agent记忆方案」，而在于它揭示了一个关键的工程与认知命题：「最优经验未必来自最强教师，而来自与自身策略最匹配的自我经验」。这一「认知对齐」现象对整个Agent工程体系有深远影响——它意味着未来Skill生态的演化路径将从「人类工程师集中供给」转向「Agent分布式自生长」，Skill的质量评估标准也将从「功能完备性」转向「与特定Agent策略的适配度」。对从业者的务实启示是：在设计Agent系统时，应优先构建「轨迹采集-离线蒸馏-动态评分-RL内化」的完整数据闭环，而非单纯堆叠外部Skill数量；复合奖励函数的设计（区分结果与过程）是让Agent习得「元能力」的关键工程杠杆，值得在生产级Agent系统中优先落地验证。
**内容简介：**

- **核心问题与动机：** 当前Agent的Skill主要依赖人类编写与维护，存在增长瓶颈、质量参差不齐、与Agent自身策略不匹配三大痛点，EvolveR旨在让Agent从自身交互轨迹中自动蒸馏可复用「认知Skill」。
- **技术路径与系统设计：** EvolveR构建了一个四阶段闭环生命周期：
- 「在线交互」：Agent调用外部知识库与内部经验库执行任务，生成完整轨迹；
- 「离线自蒸馏」：冻结参数后回顾成败轨迹，提炼简洁经验策略；
- 「经验库维护」：对新经验进行语义去重、合并与动态评分，低分经验自动剪枝；
- 「策略进化」：通过GRPO强化学习，让模型内化「何时检索、检索什么、如何应用经验」的元能力。
- **奖励函数设计：** 采用复合奖励机制，区分结果奖励（答案精确匹配）与格式奖励（推理步数适度、搜索多样性），避免稀疏奖励导致Agent无法学习中间行为。
- **关键实验发现：** 在7个复杂问答基准上，EvolveR在Qwen2.5-3B/7B上均超越CoT、RAG、SFT、Search-R1等基线；尤为重要的是，模型规模达到3B后，Agent自蒸馏的经验效果优于GPT-4o-mini教师总结，揭示「认知对齐」现象。
- **研究团队：** 上海人工智能实验室KnowledgeXLab团队，论文已被ICML 2026接收。

**核心洞察：** SxS的真正意义在于将「流式交互体验」从工程/前端问题升维为「模型可学习的披露策略」问题。这一转变揭示了一个重要的产品与训练范式：优化目标不应是TTFT（首token延迟）这类系统指标，而应是「第一个有依据的有用内容何时出现」。对推理模型的后训练而言，这意味着需要同时训练两个维度——「想得对」（推理准确率）与「说得对时机」（披露节奏），二者可以通过蕴含对齐数据+RL联合优化。对产品与工程团队的启示是：在推理型产品的用户体验设计中，「内容延迟」比「响应延迟」更接近用户真实感知，而SxS提供了一条无需改架构、仅靠后训练数据与RL即可落地的路径——但需警惕SFT-only的准确率陷阱，RL恢复阶段才是真正的成本所在。
**内容简介：**

- **核心问题定义**：传统单流自回归接口下，模型生成token既是状态更新又是不可撤回的公开承诺，导致「沉默税」——要么长时间无输出，要么过早承诺引发错误前缀锁定。
- **方法核心——SxS Interleaved Reasoning**：提出「并列式交错推理」，在同一自回归上下文中引入两类动作：`think`（内部推理，不对用户可见）与`speak`（仅在当前推理前缀支持时才披露的用户可见内容），实现「边想边说」的可控可见性生成。
- **训练三阶段**：① 构造「蕴含对齐的交错轨迹」，仅将被推理前缀支持的答案片段放入speak；② SFT阶段让模型掌握双动作格式语义；③ GRPO强化学习阶段用结果正确性信号恢复推理准确率，避免SFT-only带来的性能下滑。

- **部署友好性**：无需改变模型架构，依赖数据构造+SFT+RL即可在标准自回归模型上实现披露策略学习。

> **分类：** 观点洞察

**内容简介：**

- 事件背景：上海交大一名学生利用AI工具伪造报销收据，私吞奖金，事件曝光后引发广泛关注。

**核心洞察：** FedRE的核心工程价值在于：通过「纠缠表征」这一轻量级设计，将原本需要在「数据量-隐私-带宽」三个维度做硬取舍的联邦学习问题，转化为一个「单向量上传 + 软标签监督」的优雅解法。其可复用打法在于：用「信息混淆」替代「信息压缩」来实现隐私保护，同时利用跨类别软监督信号改善分类器泛化能力，两个目标在同一操作中协同实现，而非相互对冲。对工程落地团队的启示是：在设计隐私保护方案时，不必局限于差分隐私或同态加密等重型方案，「表征层的语义混淆」可能是一条在推理成本、通信效率与合规强度之间取得更优平衡的工程路径，尤其适用于金融、医疗等数据孤岛场景下的多机构联合建模。
**内容简介：**

- **研究背景与核心问题**：联邦学习在实际部署中面临「模型性能、数据隐私、通信开销」三者难以兼顾的困境，尤其在客户端采用不同架构（如ResNet vs ViT）的模型异构场景下，传统FedAvg直接参数聚合的方式完全失效。
- **现有方法的局限**：
- FedAllRep：上传全量表征，性能好但通信开销大、易遭表征逆向攻击泄露隐私。
- FedGH：用类别原型替代，通信与隐私有所改善，但决策边界过于尖锐，模型性能下降。
- **FedRE核心机制**：提出「纠缠表征」作为新型客户端知识表示——每轮通信中，客户端将不同类别的本地表征以随机权重融合为一个纠缠表征，并生成对应纠缠标签编码，仅上传该单一向量至服务器训练全局分类器。
- 随机权重每轮重采样，增加多样性，使分类器学习更平滑的决策边界。
- 多类别信息融合大幅提升逆向攻击重建难度，隐私保护显著增强。
- 每客户端仅上传一个向量，上传阶段通信开销最低。
- **实验验证**：在模型异构设置下，FedRE性能优于FedGH；隐私评估中纠缠表征重建结果几乎不可辨识；通信开销在上传阶段为对比方法中最低。
- **适用范围**：同时支持模型异构与模型同构两类联邦学习场景，具备较强通用性。
